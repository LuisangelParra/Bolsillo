# CLAUDE.md — Bolsillo (iOS)

Platform-specific agent guidance. Obey root `../CLAUDE.md` and `../.specify/memory/constitution.md` first.

## Stack
- Swift 5.10, SwiftUI, deployment target iOS 17.
- Xcode project is **generated by Tuist** from `Project.swift` (committed, reproducible) — do not hand-edit `Bolsillo.xcodeproj`.
- Local code lives in the SPM package `Packages/Bolsillo` (`Package.swift`, `defaultLocalization: "es"`).
- Persistence wired but not yet used: GRDB.swift 6.29+ (SQLCipher build enabled when schema lands).
- Lint: SwiftLint (`.swiftlint.yml`).

## Package / target layout
Clean architecture, one SPM library product per layer:
- `BolsilloDomain` — value types + ports only, no dependencies (`Money`, `Currency`, `Transaction`, `Ports`).
- `BolsilloData` — port implementations + seeds; depends on `BolsilloDomain` + GRDB.
- `FeatureRecord` — fast expense-recording UI; depends on `BolsilloDomain`.
App targets (in `Project.swift`): `Bolsillo` (app, `App/Sources/**`) and `BolsilloTests`.
Dependency direction: FeatureRecord → BolsilloDomain ← BolsilloData. AI sits behind the classifier port in `BolsilloDomain`.

## Build & test
Run from `ios/`:
- `swiftlint --config .swiftlint.yml` — lint.
- `tuist generate --no-open` — regenerate the Xcode project (run after editing `Project.swift`).
- `xcodebuild test -workspace Bolsillo.xcworkspace -scheme Bolsillo -destination 'platform=iOS Simulator,name=iPhone 16,OS=latest' CODE_SIGNING_ALLOWED=NO` — build + test.
- Fast domain loop: `swift test` from `Packages/Bolsillo/` (host build via the `.macOS` floor).

## Money rule (non-negotiable)
Money is **integer minor units** via the `Money` struct (`minorUnits: Int`). NEVER `Double`/`Float` for money — no floating-point storage or accessor exists on `Money`, keep it that way. Minor-unit count per currency comes from `Currency.decimalDigits`.

## Localization rule
Default locale `es` (`defaultLocalization`/`developmentRegion`); `en` switchable. NO hard-coded user-facing strings — every string lives in `App/Resources/Localizable.xcstrings` with es + en entries.

## Source of truth
The spec in `specs/NNN-feature-name/` (`spec.md` + `plan-ios.md` + `tasks-ios.md`) governs. If code and spec disagree, fix the spec first, then the code.
